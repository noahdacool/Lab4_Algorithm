--- PaintDriver.cpp ---

global constants

  • path for file operations (I will use .\ for this project for portability, but a different path such as .\database is generally a good idea to avoid overwriting program files).

  • important numbers:

    • square feet per gallon of paint

    • cost per gallon of paint

    • cost per coat / hour of labor

    • cost per mile of transit

  • headers, prompts, and other messages (grouped for easy editing and uniformity)

main

  • prompts the user for client name, distance to house, and maximum number of rooms with initHouse, then creates the House object

  • repeatedly displays a menu and gets user input with getMenuInput and passes the input into directControl until the user enters Q

directControl

  • accepts any char and a House object by reference

  • uses a switch on the char to pass the House object into the correct function (addRoom, listRooms, or quitWithInvoice) or display an error

getMenuInput

  • displays the menu with displayMenu, then gets and returns a char from the user with the getSafeInput function from myutilities.h

displayMenu

  • displays the menu options A - Add room, L - List rooms, and Q - Quit and prepare invoice

initHouse

  • gets name, distance, and maximum values from the user using getline function and the getPositiveInput function from myutilities.h

  • creates and returns the House object by passing the user input into the three-argument constructor

addRoom

  • exits immediately with an error message if the max rooms has been reached, found by calling House.getNumRooms and House.getMaxRooms

  • gets the length, width, height, and number of coats for the room from the user with the getPositiveInput function from myutilities.h

  • adds a new room to the House object by calling House.addRoom with the user input

listRooms

  • displays stored and calculated information about the house by calling House.showRooms

quitWithInvoice

  • prints the invoice to the screen by calling generateInvoice with cout

  • asks the user whether to save the file with the getValidYN function from myutilities.h

  • saves the invoice to a file with the saveInvoice function if the user desires

saveInvoice

  • gets a valid filename (not file path) from the user with the getValidFilename function from myutilities.h

  • saves the file by calling the generateInvoice function with the ostream object

  • repeats the process if the file failed to open

generateInvoice

  • calculates the costs and totals for the invoice

    • the labor charge is the cost per hour times the total coats for all rooms

    • the number of cans for each room is the wall area divided by the square feet per gallon, rounded up

    • the product charge for each room is the cost per gallon times the number of cans

    • the total product charge is the sum of the product charges for each room

    • the travel charge is the cost per mile times the distance to the house

    • the final total is the labor charge plus the total product charge plus the travel charge

  • prints the stored information about the house: client name, distance, max rooms, number of rooms

  • prints the labor charge in these columns: number of rooms, number of coats, hourly rate, total

  • prints the product charge in these columns: room number, cans needed, cost per can, total

  • prints the subtotal of all product charges

  • prints the travel charge in these columns: distance, cost per mile, total

  • prints the grand total

--- myutilities.h / myutilities.cpp ---

The input error-checking and output formatting tools are my own. Here are their descriptions.

numToStr

This function is an alternative to setprecision from iomanip that does not affect the ostream flags It returns the given number (int, float, etc.) as a string with the given number of digits after the decimal

  • parameters: auto num, int precision

  • num is copied into a new double

  • the decimal in the double is moved precision places to the right by multiplying factors of ten

  • the double is rounded by adding 0.5 and truncating into a new int

  • the int is cast into a new string

  • leading zeros are added to the string until it has at least one more digit than precision

  • a decimal is inserted into the string leaving precision digits to the right

  • the now formatted string is returned

getSafeInput

  • parameters: auto input by reference, const char* prompt, const char* typeError

  • prompts the user for input using prompt

  • if the input fails to be extracted into input, typeError is printed and the process repeats

getPositiveInput

  • paramters: auto input by reference, const char* prompt, const char* valueError, const char* typeError

  • gets input from the user by passing input, prompt, and typeError into getSafeInput

  • if the input <= 0, valueError is printed and the process repeats

getValidYN

  • paramters: char input by reference, const char* prompt, const char* valueError, const char* typeError
  • gets input from the user by passing input, prompt, and typeError into getSafeInput
  • if the input is not equal to Y, y, N, or n, valueError is printed and the process repeats

getValidFilename

  • paramters: const char* prompt, const char* valueError

  • gets input frmo the user with prompt and getline

  • if the input does not match the regular expression [\w\s\.-]*, valueError is printed and the process repeats

  • returns a string with numbers, letters, periods, spaces, dashed, and underscores (no file paths)

center

  • parameters: string text, int length, and char fill default to ' '

  • adds an equal amount of fill on either side of text until it returns a string with length characters

  • the right side may be given one more fill character to account for an odd difference between text and length

  • if text is too long, it will instead be truncated to a string with length characters that ends in ...

l_just

This function is an alternative to setw and left and setfill from iomanip It has the added safeguard of truncating text that is too long

  • parameters: string text, int length, and char fill default to ' '

  • adds fill characters to the right of text until it returns a string with length characters

  • if text is too long, it will instead be truncated to a string with length characters that ends in ...

r_just

This function is an alternative to setw and right and setfill from iomanip It has the added safeguard of truncating text that is too long

  • parameters: string text, int length, and char fill default to ' '

  • adds fill characters to the left of text until it returns a string with length characters

  • if text is too long, it will instead be truncated to a string with length characters that ends in ...

d_just

  • parameters: string textL, string textR, int length, and char fill default to ' '

  • adds fill characters between textL and textR until it returns a string with length characters

  • if textL and textR combined would be longer than length, something will be truncated:

    • if textR is shorter than two-thirds of length, textL will be truncated so both will fit with ... in between

    • if textL is shorter than two-thirds of length, textR will be truncated so both will fit with ... at the end

    • otherwise textL and textR will each be truncated to a string half the size of length with ... at the end
